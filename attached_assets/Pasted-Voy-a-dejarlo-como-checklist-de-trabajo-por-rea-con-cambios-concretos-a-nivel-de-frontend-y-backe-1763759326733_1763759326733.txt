## 1. Carga de fotografías de nuevos garments

### 1.1. Frontend – `curator-new-garment`

1. Guardar también el `File`, no solo el base64

* Añadir estado:

```ts
const [photoFile, setPhotoFile] = useState<File | null>(null);
```

* Modificar `handlePhotoChange`:

```ts
const handlePhotoChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  const file = e.target.files?.[0];
  if (!file) return;

  setPhotoFile(file); // guardar el File para el envío

  const reader = new FileReader();
  reader.onloadend = () => {
    setPhotoPreview(reader.result as string); // solo para preview
    // opcional: ya no seteas photoUrl en el form, lo dejarás al backend
    form.setValue("photoUrl", ""); 
  };
  reader.readAsDataURL(file);
};
```

2. Cambiar el submit a `FormData`

* En la `mutation` de creación de garment, en lugar de llamar a `apiRequest` con JSON:

```ts
const createGarmentMutation = useMutation({
  mutationFn: async (data: FormValues) => {
    const formData = new FormData();
    formData.append("code", data.code);
    formData.append("size", data.size);
    formData.append("color", data.color);
    formData.append("gender", data.gender);
    formData.append("status", data.status ?? "IN_STOCK");
    formData.append("categoryId", data.categoryId);
    formData.append("garmentTypeId", data.garmentTypeId);
    formData.append("collectionId", data.collectionId);
    if (data.lotId) formData.append("lotId", data.lotId);
    if (data.rackId) formData.append("rackId", data.rackId);

    if (photoFile) {
      formData.append("photo", photoFile); // nombre de campo EXACTO "photo"
    }

    const res = await fetch("/api/garments", {
      method: "POST",
      headers: {
        ...getAuthHeaders(), // importante: NO poner Content-Type aquí
      },
      body: formData,
    });

    if (!res.ok) {
      const text = await res.text();
      throw new Error(text || res.statusText);
    }

    return res.json();
  },
});
```

> Nota: no uses `Content-Type: application/json` cuando envías `FormData`.

### 1.2. Backend – ruta `POST /api/garments`

3. Asegurar que Multer se aplique siempre que sea multipart

La estructura actual ya funciona si llega `multipart/form-data` con campo `photo`. Solo confirmaría:

* Que el middleware:

```ts
if (req.is('multipart/form-data')) {
  return upload.single("photo")(req, res, next);
}
```

se ejecuta antes del handler principal.

* No hace falta cambiar la lógica de:

```ts
const photoUrl = req.file
  ? `/uploads/${req.file.filename}`
  : (bodyPhotoUrl || null);
```

porque ahora `req.file` sí se llenará y `photoUrl` quedará como `/uploads/...`.

### 2.2. Frontend – parsear bien el error en `apiRequest`

4. Modificar `throwIfResNotOk` para intentar leer JSON y extraer `message`:

```ts
async function throwIfResNotOk(res: Response) {
  if (!res.ok) {
    let message = res.statusText;
    try {
      const data = await res.json();
      message = data.message || message;
    } catch {
      const text = await res.text();
      if (text) message = text;
    }
    throw new Error(message);
  }
}
```
